내가 지금 공부중인 프로그래밍 언어는 자바다. 자바는 작성일인 현재를 기준으로도 이미 역사가 꽤 쌓인 언어라서, 자바를 언급하면 12가지 수식어가 꼭 따라오곤 한다.
>Java is a **simple**, **object-oriented**, **distributed**, **interpreted**, **robust**, **secured**, **architecture neutral**, **portable**, **high-performance**, **platform independent**, **multi-threaded**, and **dynamic** language.

개인적으로 정리해보면서, 몇 특징은 완전히 이해되진 않았고, 어떤 특징들은 같은 특징을 굳이 다른 표현으로 나누어 놓은 것 같길래 하나로 합쳐버렸다. 그러면 자바의 12가지같은 9가지 대표적인 특징에 대해 알아보자.

## 1. 간단하다 (Simple)
자바는 배우기 매우 쉽고 문법이 간단하고 깔끔하여 이해하기 쉽다. Sun Microsystem에 따르면, 다음과 같은 이유로 자바가 간단한 프로그래밍 언어임을 설명하고 있다.
- 자바 문법은 C++을 기반으로 한다(C++을 먼저 접한 개발자들이 배우기 쉬움)
- 자바는 복잡하고 거의 사용되지 않는 많은 기능(예: 명시적 포인터, 연산자 오버로딩 등)을 제거했다.
- 자바에는 가비지컬렉션 기능이 있기 때문에 참조되지 않은 객체를 제거할 필요가 없다.

## 2. 객체 지향적이다 (Object-oriented)
자바는 객체 지향 프로그래밍 언어이다. 자바에서 원시 자료형을 제외한 모든 것을 객체라고 불러도 무방하다. 객체 지향은 데이터와 동작을 모두 통합하는 다양한 유형의 객체 조합으로 소프트웨어를 구성하는 것을 의미한다.

## 3. 플랫폼 독립적이다 (Platform Independent)
인터프리터 언어이다(interpreted), 아키텍쳐 중립적이다(architecture neutral), 이식이 쉽다(portable)라는 특징 모두 플랫폼 독립적이라는 특징에서 파생한다.
   
자바는 컴파일 언어이면서 인터프리터 언어이다. 그 이유는 자바의 실행 과정을 통해 알 수 있다. 자바는 먼저 텍스트로 된 소스 코드를 컴파일하여 이진 파일(클래스 파일)로 만든 다음, 자바 런타임이 이 클래스 파일을 인터프리트(interpret)하면서 실행한다. 이렇게 자바는 컴파일 언어에 가까운 속도와 함께 플랫폼/아키텍쳐로부터 독립성을 얻은 것이다.

이러한 자바의 장점을, "Write once, run everywhere." 라고 칭찬하곤 하는데, 개인적으로는 JVM(Java Virtual Machine, 자바 가상 머신) 없으면 실행되지 않는 자바가 과연 독립적이라고 봐야 하는지 의문이다.

## 4. 안전하다 (Secured)
자바의 보안이 뛰어난 이유는 다음과 같다.
- 명시적 포인터가 없다.
- 자바 프로그램은 가상 머신 샌드박스 내에서 실행된다.
- ClassLoader(클래스로더): 자바의 클래스 로더는 자바 클래스를 JVM에 동적으로 로드하는 데 사용되는 JRE(Java Runtime Environment, 자바 런타임 환경)의 일부이다. 로컬 파일 시스템의 클래스에 대한 패키지를 네트워크 소스에서 가져온 패키지와 분리해서 보안을 추가한다.
- Bytecode Verifier(바이트코드 검증기): 객체에 대한 엑세스 권한을 침해할 수 있는 잘못된 코드가 있는지 코드를 확인한다.
- Security Manager(보안 관리자): 클래스가 액세스할 수 있는 리소스를 결정한다.

## 5. 강력하다 (Robust)
프로그래밍 언어가 강력하다고 할 때 보통 robust라고 표현하는 것 같다. 자바가 강력하다고 평가받는 이유는 다음과 같다.
- 메모리 관리 기능이 탁월하다.
- 포인터의 개념이 없어서 보안 문제를 쉽게 해결해준다.
- 자바는 자바 프로그램에서 더 이상 사용하지 않는 객체를 제거하기 위해 JVM에서 실행되는 가비지컬렉션을 제공한다.
- 자바는 예외 처리가 강제적이고, 자료형 검사가 엄격하다.

## 6. 동적이다 (Dynamic)
자바는 동적 언어이다. 이는 자바 실행 시점(runtime)에 필요한 클래스와 객체를 동적으로 불러올 수 있음을 의미한다. 다시말해, 프로그램 실행 도중에 새로운 기능을 추가하거나 변경할 수 있게 한다. 이는 자바 개발자들에게 유연한 앱 개발을 가능하게 한다.

## 7. 고성능이다 (High-performance)
자바는 자바 바이트코드가 네이티브 코드와 가깝기 때문에 다른 전통적인 인터프리터 언어보다 속도가 빠르다. 그래도 여전히 C++같은 전형적인 컴파일언어보다는 약간 느리다. 따라서 속도는 인터프리터 언어<자바<컴파일 언어 순으로 보면 되겠다.

## 8. 멀티 스레드를 지원한다.
스레드는 동시에 실행되는 별도의 프로그램과 같다. 멀티스레드를 정의하여 한 번에 많은 작업을 처리하는 자바 프로그램을 작성할 수 있다. 멀티 스레드의 주요 장점은 스레드들이 각자의 메모리를 차지하지 않고, 공통 메모리 영역을 공유한다는 것이다.

## 9. 분산성이 있다.
자바는 사용자가 자바로 분산 응용 프로그램을 쉽게 만들 수 있도록 하기 위해 배포된다. RMI와 EJB는 분산 응용 프로그램을 만드는 데 사용된다. 자바의 이 기능을 사용하면 인터넷의 모든 시스템에서 메소드를 호출하여 파일에 엑세스할 수 있다.

- RMI(Remote Method Invocation, 원격 메소드 호출): 자바 프로그램에서 각 객체간, 컴퓨터간 메서드를 호출할 수 있게 해주는 기술
- EJB(Enterprise Java Beans, 자바 엔터프라이즈 빈즈): 분산형 자바 어플리케이션을 개발하기 위한 서버측 컴포넌트 모델델
