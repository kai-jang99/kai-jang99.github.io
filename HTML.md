> HTML은 웹 페이지의 구조와 내용을 정의하는 마크업 언어입니다.

1. 용도: 문서의 구조를 설정하여 웹페이지를 구성

2. 역사
    1. 1991년 Tim Berners-Lee에 의해 제안
    2. 현재 HTML5 사용

3. 작성 도구
    1. Visual Studio Code
    2. Atom (맥os 용)
    ...

4. 참조 사이트
    - https://developer.mozilla.org/ko/docs/Web/HTML
    - https://www.w3schools.com/

5. 플러그인 설치
    1. Live Server: 실시간으로 HTML파일을 웹브라우저에 반영
    2. Auto Rename Tag: 태그를 자동으로 닫아주고 이름 변경 시 자동 변경

6. 웹 컨텐츠 작성 시 고려할 점
    1. 웹 표준(크로스 브라우징): 모든 브라우저에서 동일하게 보이도록 작성
    2. UX에 기반

7. 문법
    1. 태그의 기본 구조(Element)
        1. `<열린 태그명>컨텐츠</닫힌 태그명>`
        2. `<열린 태그명 속성="값" 속성="값"...>컨텐츠</닫힌 태그명>`
        3. `<열린 태그명></닫힌 태그명> 또는 <태그명 />` <- 빈 태그

```
<!DOCTYPE html>
<html> <!-- root tag -->
	<head>
		메타 정보나 선언부
	</head>
	<body>
		화면에 보여질 본문 내용
	</body>
</html>
```
8. 주석 `<!-- 주석 내용 -->`
    1.  `<head>`
	    1. `<meta>`
        2. `<title>`
	2. 글자 꾸미기
	3. 이미지
		1. `<img src="이미지 경로" alt="대체 텍스트"/>`
		2. 이미지 포맷 종류와 특징
            - gif 
                비손실 압축
                8비트 색상만 지원
            - jpg(jpeg)
                뛰어난 압축률(작은 용량)
                여러번 반복 저장하면 손실율이 매우 높아진다
            - png
                gif의 대체 포맷
                비손실 압축
                투명도 지원
            - webp
                gif, jpg, png를 모두 대체할 수 있는 이미지 포맷
        3. 경로(path)
            1. 상대 경로: 현재 문서를 기준으로 한 경로
            2. 절대 경로: 루트 디렉토리를 기준으로 한 경로
                - 로컬(실제) 경로: 실제 물리적인 경로
                - 가상 경로: URL
		4. 하이퍼 링크
            1. `<a href="이동할 경로(URL)">링크 텍스트</a>`
            2. URL(Uniform Resource Locator)
                - 네트워크 상에서 자원이 어디에 있는지 알려주기 위한 규약
				- 형식
                    `프로토콜 명 ://도메인(아이피 주소):포트번호/폴더명/파일명#세부위치`
                        https://www.naver.com:80/index.html
                        포트번호: 세부주소
                        세부위치: 페이지 내 바로 이동
	4. 테이블, div, span
	5. input type
